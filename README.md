# GRC Program Starter Kit
**Company: MedBridge Supply Co.**

A practical governance, risk, and compliance (GRC) portfolio project built around a mid-size healthcare supply chain technology company. MedBridge Supply Co. operates a SaaS eProcurement platform used by hospitals, long-term care facilities, and healthcare networks to manage medical supply purchasing (~300 employees, Milwaukee, WI).

This project demonstrates hands-on GRC documentation skills aligned to industry frameworks including **NIST CSF**, **NIST 800-171**, **ISO 27001**, and **SOC 2 Trust Services Criteria**.

---

## Repository Contents

| File | Description | Framework |
|------|-------------|-----------|
| `nist-csf-gap-assessment.xlsx` | Current vs. target maturity assessment across all five NIST CSF functions | NIST CSF |
| `control-mapping.xlsx` | Cross-reference of NIST CSF controls to ISO 27001 Annex A and SOC 2 TSC | NIST CSF / ISO 27001 / SOC 2 |

## Frameworks Referenced

- **NIST Cybersecurity Framework (CSF) 2.0** — Primary governance and risk framework
- **NIST SP 800-171** — Controls for protecting Controlled Unclassified Information (CUI)
- **ISO/IEC 27001:2022** — Information security management system standard
- **SOC 2 Trust Services Criteria** — Audit and compliance framework for SaaS companies

---

## How to Use This Repository

Each artifact is designed to stand alone or be used together as part of a unified GRC program build. Recommended sequence:

1. Start with the **NIST CSF Gap Assessment** to establish baseline maturity
2. Use the **Risk Register** to document and prioritize identified risks
3. Reference the **Control Mapping** to align remediation efforts across frameworks
4. Use the **POA&M Template** to track remediation commitments and timelines
5. Deploy the **Vendor Risk Questionnaire** during third-party onboarding or annual reviews
